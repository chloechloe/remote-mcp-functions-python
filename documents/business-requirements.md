# 映画館の窓口業務エージェント 要件定義書

## 1. 文書の目的
本書は、Azure Functions 上で動作する MCP サーバーに対し、映画館の窓口業務を行うエージェント向けの MCP ツール群を追加するための業務要件を定義するものである。

本プロジェクトは PoC であるため、実運用に必要な厳密性よりも、窓口業務の主要な流れを一通り再現できることを優先する。

## 2. 前提
- 参照対象として指定された [documents/operation-scenario.md](documents/operation-scenario.md) を確認したが、現時点では内容が記載されていない。
- そのため、本書は利用者から提示された「映画館の窓口業務を行うエージェント」という目的に基づき、PoC として妥当な範囲の要件を定義する。
- 本要件は、MCP クライアントから呼び出されるツール群を前提とし、対話型エージェントが窓口担当者の代替として案内・受付・発券判断を行えることを目指す。

## 3. 想定業務スコープ
本 PoC における窓口業務の対象範囲は以下とする。

- 上映作品の案内
- 上映スケジュールの案内
- 空席状況の確認
- チケット予約の受付
- 予約内容の確認
- 予約の変更
- 予約の取消
- 座席案内
- 劇場案内・注意事項案内

以下は PoC の対象外、または簡易化対象とする。

- 実決済連携
- 会員ポイント管理
- クーポン・複雑な割引条件
- 複数館横断の在庫統合
- 紙チケット印刷機やゲート機器との実連携
- 厳密な同時予約制御

## 4. 利用者
### 4.1 窓口利用者
映画館で作品情報の確認、座席確認、予約、変更、取消を行う来場者。

### 4.2 窓口業務エージェント
MCP ツールを利用して来場者対応を行う対話エージェント。利用者の意図を理解し、必要なツールを呼び出して業務を完了させる。

### 4.3 劇場管理者
上映作品、上映回、料金、座席、予約情報を管理する運営側の担当者。PoC では管理用 UI は必須ではなく、管理データは簡易なデータソースで保持してよい。

## 5. 業務要件一覧

### 機能1: 作品案内
#### 目的
来場者に対して、上映中または上映予定の作品情報を案内する。

#### 要件
- エージェントは作品一覧を取得できること。
- エージェントは作品名、上映時間、年齢制限、ジャンル、概要を案内できること。
- エージェントは上映中、上映予定などの状態で絞り込めること。
- エージェントは作品名の部分一致や曖昧な問い合わせに対して候補を提示できること。

#### 管理データ
- 作品ID
- 作品名
- 作品概要
- 上映時間
- ジャンル
- 年齢制限
- 公開開始日
- 公開終了日
- 上映状態
- 音声・字幕属性

### 機能2: 上映スケジュール案内
#### 目的
来場者が希望する日付・作品に対する上映回を確認できるようにする。

#### 要件
- エージェントは日付と作品を条件に上映回を検索できること。
- エージェントは上映開始時刻、終了予定時刻、スクリーン番号を案内できること。
- エージェントは本日、明日、日付指定などの条件を扱えること。
- エージェントは該当上映回がない場合、代替日や近い時間帯を提示できること。

#### 管理データ
- 上映回ID
- 作品ID
- 上映日
- 開始時刻
- 終了時刻
- スクリーンID
- 上映方式
- 販売状態

### 機能3: 空席照会
#### 目的
来場者が予約前に空席の有無を確認できるようにする。

#### 要件
- エージェントは上映回単位で空席数を取得できること。
- エージェントは空席あり / 残りわずか / 満席 などの簡易状態を案内できること。
- エージェントは座席表ベースで空席座席を取得できること。
- エージェントは希望条件に応じて候補座席を提示できること。

#### 管理データ
- 上映回ID
- 座席ID
- 座席状態
- 仮押さえ有無
- 予約ID

### 機能4: 座席案内
#### 目的
来場者の希望に合う座席を案内する。

#### 要件
- エージェントはスクリーンごとの座席表を取得できること。
- エージェントは通路側、中央付近、前方、後方などの簡易条件で候補を抽出できること。
- エージェントは連番座席を優先して提案できること。
- エージェントは車椅子席など特別席を識別できること。
- エージェントは既に埋まっている座席を除外できること。

#### 管理データ
- 座席ID
- スクリーンID
- 列番号
- 番号
- 座席種別
- 位置属性
- 利用可否

### 機能5: チケット予約受付
#### 目的
来場者の希望内容に基づき予約を登録する。

#### 要件
- エージェントは作品、上映回、人数、座席、代表者情報を受け付けられること。
- エージェントは予約登録前に入力内容を確認表示できること。
- エージェントは予約完了時に予約番号を返却できること。
- PoC では決済未完了でも予約登録を可能とするか、支払状態を仮状態で保持できること。
- エージェントは満席や不正座席指定時に代替候補を案内できること。

#### 管理データ
- 予約ID
- 予約番号
- 上映回ID
- 作品ID
- 来場者名
- 連絡先
- 予約人数
- 予約座席一覧
- 予約状態
- 受付日時
- 支払状態

### 機能6: 予約確認
#### 目的
来場者が既存予約の内容を確認できるようにする。

#### 要件
- エージェントは予約番号を用いて予約を検索できること。
- エージェントは予約内容として作品名、上映日時、座席、人数、状態を表示できること。
- エージェントは予約が存在しない場合にその旨を返却できること。
- エージェントは個人情報の過剰表示を避け、必要最小限の情報のみ案内すること。

#### 管理データ
- 予約番号
- 予約状態
- 上映回情報
- 座席一覧
- 代表者情報

### 機能7: 予約変更
#### 目的
来場者の予約内容を変更できるようにする。

#### 要件
- エージェントは予約番号を指定して既存予約を取得できること。
- エージェントは上映回変更、座席変更、人数変更の一部または全部を受け付けられること。
- エージェントは変更後の空席状況を確認した上で更新できること。
- エージェントは変更不能な条件に対して理由を返却できること。
- PoC では変更履歴は簡易記録でよい。

#### 管理データ
- 変更対象予約ID
- 変更前内容
- 変更後内容
- 変更日時
- 変更理由
- 更新者種別

### 機能8: 予約取消
#### 目的
来場者の予約を取消できるようにする。

#### 要件
- エージェントは予約番号を指定して取消可否を確認できること。
- エージェントは取消実行後に予約状態を取消済みに更新できること。
- エージェントは取消により座席在庫を空席へ戻せること。
- PoC では払戻計算は簡易化または対象外としてよい。

#### 管理データ
- 予約ID
- 取消日時
- 取消状態
- 取消理由
- 払戻有無

### 機能9: 劇場案内・注意事項案内
#### 目的
来場者に劇場利用上の基本情報を案内する。

#### 要件
- エージェントは営業時間、所在地、アクセス、館内ルールを案内できること。
- エージェントは飲食可否、入場開始時刻、年齢制限注意、遅刻時の案内方針などを返却できること。
- エージェントは FAQ 的な問い合わせに対して定型回答を返却できること。

#### 管理データ
- 劇場ID
- 劇場名
- 営業時間
- 所在地
- アクセス情報
- 注意事項一覧
- FAQ 一覧

## 6. 共通業務要件
### 6.1 対話処理
- エージェントは来場者の自然言語入力から意図を判定し、適切なツールを選択できること。
- 情報不足時は不足項目のみを追加質問できること。
- エージェントは予約確定や取消確定のような重要操作前に確認を取ること。

### 6.2 エラー時の振る舞い
- 対象データが存在しない場合、分かりやすい文言で通知すること。
- 満席、予約番号不正、入力不足などの代表的なエラーを区別して返却できること。
- エージェントは業務継続可能な代替案をできるだけ提示すること。

### 6.3 PoC としての簡易要件
- データ保存先は簡易なファイル、インメモリ、または軽量ストレージでもよい。
- 認証認可は必須要件としないが、管理操作との分離は考慮すること。
- 厳密な排他制御は必須ではないが、同一座席への重複予約を極力防ぐ設計が望ましい。

## 8. 機能とデータ構造の対応
- 作品案内は `Movie` を参照する。
- 上映スケジュール案内は `ShowTime` と `Movie` を参照する。
- 空席照会と座席案内は `Seat`、`Screen`、`SeatInventory` を参照する。
- 予約受付、確認、変更、取消は `Reservation` を中心に `ShowTime`、`SeatInventory` を参照または更新する。
- 劇場案内は `TheaterInfo` を参照する。
